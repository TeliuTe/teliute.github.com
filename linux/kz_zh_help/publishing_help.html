用 KompoZer 发布网页

译者：TeliuTe 来源：基础教程网

如果页面仅保存在您的本地硬盘，只有您可以查看这个 HTML 页面，KompoZer 允许您 发布网页到一个被称作网站服务器的远程计算机上

当你发布页面到网站服务器上，KompoZer 复制(上传)您的页面到一个可以让其他人访问 的计算机上。大多数 ISP 提供网站空间用于发布网页。要查找发布网页的网站服务器，可以询问您的 ISP 或是系统管理员。

发布文档

要发布一个文档:

  1. 打开您想发布的 HTML 文档，或者新建一个 KompoZer 文档。
  2. 当你准备发布文档到远程计算机时，点击发布按钮 。
    • 如果您之前已经发布过这个文档， KompoZer 会记住文档的发布设置，立即开始发布， 在发布过程中，KompoZer 会显示一个发布进度条。
    • 如果您之前还未发布过这个文档， KompoZer 会在发布对话框里显示设置标签， 以便您能够输入发布此文档的远程信息。更多信息请参阅: 发布页面 - 设置。当您完成信息输入，点发布即可。
  3. 要浏览您的发布页面，点击浏览按钮，测试页面链接确保没有缺失的图片。
  4. 可能您会继续编辑这个页面，当您编辑好要更新远程页面时，点击发布按钮即可。

当您首次发布文档时，KompoZer 会更改文档里的 file:/// 地址为一个 http:// 地址，指示您正在编辑的是要发布文档。如果您想保存文档到本地 (在您的计算机硬盘上)，点保存按钮，您将会看到一个提示，输入文件名和保存位置。

防止断开链接或缺失图片的小提示

  • 确保 KompoZer 文件名是以 .html 或 .htm 结尾的扩展名。 确保图片文件名是以 .JPG, .GIF, 或 .PNG 结尾的扩展名。 不要使用空格或其他特殊符号(如汉字)做文件名。 保持您的文件名较短，并且只用小写或大写字母和数字。
  • 如果您在网站上浏览您发布的页面时，发现图片没有出现，或是一个出现错误标记， 这可能是您忘了发布包含的图片。打开文件菜单，选择 发布为 命令，显示发布对话框，在 发布标签，确保选中了“包括图片和其他文件”然后点发布按钮。

更多错误提示，请参阅 解决常见发布错误.

上传已发布的文档

要上传一个已发布的文档:

  1. 在 KompoZer 窗口中，打开文件菜单，然后选择最近打开页面， 然后从列表中选择文档。

    另外，要想浏览您想更新的文档，在浏览器的地址栏中输入文档的 HTTP 地址(页面的网址)。

  2. 打开文件菜单，打开要编辑页面。
  3. 根据需要进行编辑。
  4. 当你准备发布文档到远程计算机时，点击工具栏上的发布按钮。

小提示: 要 从网站上删除已经发布的网页或图片，您必须使用 FTP(文件传输协议)程序，如果您想创建 子目录或者重命名网站上的文件，您也必须要用 FTP 程序。问一下服务提供商，如果他们 推荐了一个特定的 FTP 程序，您一般可以在网络提供商网站的帮助和支持里，找到 FTP 的 相关信息，FTP 程序也可以在一些共享站点像 ZDNet 上找到。

更改文件名或发布位置

要更改文档的名称和发布位置:

  1. 在 KompoZer 窗口中，打开文件菜单，然后选择最近打开页面，从列表中选择这个文档。

    另外，要想浏览您想更新的文档，在浏览器的地址栏中输入文档的 HTTP 地址(页面的网址)。

  2. 打开文件菜单，打开要编辑页面。
  3. 根据需要进行编辑。
  4. 打开 KompoZer 的文件菜单，然后选择 发布为，KompoZer 在发布页面对话框 显示发布标签。
  5. 如果需要的话，输入一个不同的标题。
  6. 如果需要的话，输入一个不同的文件名。
  7. 在站点名称列表里，选择发布的位置，要设置一个新的发布位置，点击新建站点按钮， 更多相关信息，请参阅: 发布页面 - 设置
  8. 点发布保存文档到新的位置。

创建一个新的发布站点

如果您计划发布文档到几个远程位置，您可以设置 KompoZer 为每个远程位置保存 发布信息，这样在发布时，您就不必每次都要输入信息。

要创建一个新的发布站点，在 KompoZer 窗口操作如下:

  1. 打开编辑菜单，选择发布站点设置 KompoZer 显示发布设置对话框
  2. 点击 新建站点。
  3. 在“站点名称”中输入一个昵称，它代表您要发布到的站点。

    例如，如果您想要在新站点发布与“Meteor”项目相关的页面，您可以使用站点的名称 “Meteor”，站点名称可以提醒您要发布的文档类型。

  4. 在“发布地址“中，输入完整的由 ISP、系统管理员或是网站主机提供商您的地址，这 个地址必须由 ftp:// 或 http:// 开头。

    发布地址是指文档发布(上传)的位置，如果您不确定是什么，请询问您的 ISP 或系统管理员。

  5. 在“您的主页 HTTP 地址”中，输入完整的用浏览器访问页面时的地址，不要在地址中包括 文件名或子目录。

    这个地址必须以 http:// 开头，在某些情况下，这个地址跟发布地址相同，如果 您不确定是什么， 请询问您的 ISP 或系统管理员，或者保持为空。

  6. 在用户名中，输入您登录 ISP 或网站主机服务所使用的用户名。
  7. 在密码中，输入跟用户名对应的密码。
  8. 选中“保存密码”，将使用密码管理器安全保存您的密码，这样您发布页面时，不必每次都输入。
  9. 点确定完成。

更改默认的发布站点

如果您设置了多个发布站点，但您通常有一个使用最多的，您可以把这个最常用的站点作为默认站点。 KompoZer 将使用默认发布站点发布您所有的文档，除非您指定了一个别的站点。

不管您设置了多少站点，您总是可以通过在文件菜单里的 发布为 命令，发布文档到指定的站点。更多 相关信息，请参阅: 更改文件名或发布位置。

要设定默认的发布站点，在 KompoZer 窗口中，按如下操作:

  1. 打开编辑菜单，然后选择发布站点设置，KompoZer 显示发布设置对话框。
  2. 从列表中选择一个发布站点。

    如果您只有一个发布站点，KompoZer 将使用它作为默认站点。

  3. 点击 设为默认。
  4. 点确定完成您的更改。

删除一个发布站点

在KompoZer 中删除发布站点，删除发布设置，如果您以后想发布到此站点，您必须 要重新输入站点的设置。

要删除一个发布站点设置，在 KompoZer 中，操作如下:

  1. 打开编辑菜单，选择发布站点设置，KompoZer 显示发布设置对话框。
  2. 从列表中选择一个发布站点
  3. 点删除站点。

    KompoZer 只删除站点设置，站点本身并不会受到影响。

  4. 点确定完成更改。

发布设置

这一章讲述 KompoZer 的发布设置，更多 KompoZer 的常规和新页设置， 请参阅: KompoZer 首选项。

发布页面 - 发布

发布页面 - 在发布标签中，您可以指定文档发布的位置，这个设置只应用于当前文档

如果您还没有看到发布页面 - 发布标签，请按下面步骤操作:

  1. 打开文件擦掉，然后选择 发布为，出来发布页面对话框。
  2. 点击发布标签。
  • 站点名称: 列出所有您创建的发布站点，您可以选择一个发布， 要创建一个新站点，点 新建站点。
  • 页面标题: 指定文档的页面标题，它将出现在访问者浏览器的标题栏中。 文档页面标题，也出现在您的书签列表里，如果您添加了书签。
  • 文件名: 指定文档的文件名，确保文件的扩展名是 .html 或 .htm

  • 警告: 如果远程发布站点上已经 存在您要发布的同名文件，新上传的文件会替换原来已经存在的，您会被提示确认覆盖操作。

  • 发布页面到子目录: 如果此项为空，KompoZer 将发布页面到 主目录(根目录)中。如果您想发布页面到一个远程子目录中，在这里输入子目录，或者从目录中选择 一个。KompoZer 会跟踪您在这儿输入的位置，因此您可以从已经使用过的远程位置里边中 选择一个，注意子目录的名称是区分大小写的。

  • 注意: 您选择的站点子目录必须是存在的。

  • 包括图片和其他文件: 选中此项，KompoZer 将同时发布此页面链接 的所有图片和其他文件，您可以选择发布到同一位置，或者发布到已经存在的站点子目录中。

小提示: 要创建一个远程子目录，或者删除已经发布的网页或图片， 您必须使用 FTP(文件传输协议)程序，问一下服务提供商，如果他们 推荐了一个特定的 FTP 程序，您一般可以在网络提供商网站的帮助和支持里，找到 FTP 的 相关信息，FTP 程序也可以在一些共享站点像 ZDNet 上找到。

发布页面 - 设置

发布页面 - 设置标签，允许您指定您在这个发布站点上的登录信息，跟发布设置一样。 这个设置应用于当前文档和其他要发布到这个位置上的文件。

如果您还没有看到发布页面 - 设置标签，请按如下操作:

  1. 打开文件菜单，然后选择发布为，出来发布对话框。
  2. 点设置标签。
  • 站点名称: 指定您想发布站点的昵称，输入一个较短的名称 作为此发布站点的标识。
  • 发布地址: 输入 完整的由 ISP、系统管理员提供给您的地址，这个地址必须由 ftp:// 或 http:// 开头。这个名称问我作为“主机名”或“主机服务器名”。

  • 发布地址是指文档发布(上传)的位置，如果您不确定是什么，请询问您的 ISP 或系统管理员。

  • 您的主页 HTTP 地址: 指定您发布主目录的完整地址，这是您网站主目录的 网址，不要在地址中包括文件名或子目录。

  • 这个地址必须以 http:// 开头。在某些情况下，这个地址跟发布地址相同，如果 您不确定是什么， 请询问您的 ISP 或系统管理员，或者保持为空。

  • 用户名: 指定您用来登录 ISP 或网络的用户名称。
  • 密码: 指定跟用户名对应的密码。
  • 保存密码: 选中此项，将使用密码管理器加密保存您的密码，这样您发布页面时，不必每次都输入密码。

发布设置

发布设置对话框允许您创建、编辑和删除发布站点设置，也允许您设定默认发布站点。

如果您还没有看到发布设置对话框，请按如下操作:

  1. 打开编辑菜单，然后选择发布站点设置 KompoZer 显示发布设置对话框。
  • 新建站点: 允许您设定一个新的发布站点。KompoZer 添加 新的发布站点到可用站点列表中。
  • 设为默认: 设置选中的发布站点成为默认的发布站点。通常， 默认发布站点是您最常用的发布站点。所有您创建和编辑的文档会发布到这个站点，除非您 在发布页面对话框选择了其他发布站点。

  • 要发布文档到一个不同的远程位置，打开文件 菜单，然后选择 发布为，选择一个不同的发布位置。

  • 删除站点: 从 KompoZer 中删除选中的站点和它的设置。
  • 站点名称: 指定您想指向发布站点的名称。
  • 发布地址: 输入 完整的由 ISP、系统管理员提供给您的地址，这个地址必须由 ftp:// 或 http:// 开头。

  • 发布地址是指文档发布(上传)的位置，如果您不确定是什么，请询问您的 ISP 或系统管理员。

  • 您的主页 HTTP 地址: 指定您发布主目录的 HTTP 地址，不要在地址中包括文件名或子目录。

  • 这个地址必须以 http:// 开头。在某些情况下，这个地址跟发布地址相同，如果您不确定是什么， 请询问您的 ISP 或系统管理员，或者保持为空。

  • 用户名: 指定您用来登录 ISP 或网络的用户名称。
  • 密码: 指定跟用户名对应的密码。
  • 保存密码: 选中此项，将使用密码管理器安全地保存您的密码，这样您发布页面时，不必每次都输入密码。

解决常见问题

如果文件发布失败，发布状态对话框会显示一条错误信息，提示错误原因以及如何修复它。 点击错误诊断按钮，获得解决发布错误的帮助。

如果您仍然不能发布文件，打开 KompoZer 的文件菜单选择保存，保存文件到本地硬盘， 您可以稍后重试发布它，要快速打开这个文件，打开 KompoZer 的文件菜单，然后选择最近 打开页面。

验证您的发布设置

要验证您的发布设置:

  1. 关闭发布状态对话框，如果它是开着的话。
  2. 打开编辑菜单，然后选择发布站点设置。
  3. 在发布设置对话框中，检查站点设置是否正确，如果您不确定，可以询问 ISP 或主机提供商。
    • 检查您正确输入了发布设置您可能在设置中出现了错误的拼写。
    • 检查您是否正确输入了发布地址。 网站主机或 ISP 可以把发布地址 用“服务器名称”、“主机名称”或者是“服务器/ 主机”来表示。他们常常指定发布位置为 ftp.myisp.com/username，这儿的 username 是您的用户名。

      如果发布地址是正确的，您必须要加上 ftp:// 或者 http://。例如，正确的发布地址应该是这样的 ftp://ftp.myisp.com/username。

检查文件名

检查各个发布失败的文件名，确保那些文件名:

  • 仅使用数字、小写字母、大写字母，如果创建的可接受文件名是大写字母，您可能会用小写而发生错误。

    当您发布文件到网站上，文件名是大小写敏感的，记住那些只有大写或者大小写混合的文件名是很 困难的。

    例如，当你试图中浏览器中输入网页的地址文件名时，您必须输入精确的，使用大小写组合的文件名。

  • 不要使用标点符号或空格，下划线( _ ) 和连字符( - ) 是可以的。
  • 文件名结尾是 .html 或 .htm (对于 KompoZer 文件名)。
  • 不要超过32个字符。

修复发布错误

如果文件发布失败，仔细查看 KompoZer 在发布状态对话框里的错误提示。 您可以通过错误消息帮助检测错误原因以及该如何解决。

错误消息:

文件 未找到

或者

Y 的 X 发布失败

错误描述: 一个或几个图片文件或 CSS 文件发布失败，原因是 KompoZer 未能找到它们，可能的原因是:

  • 您输入的文件位置错误。
  • 网站上的文件位置不允许访问。
  • 文件位置改变，或者文件被删除或移动到其他位置。

解决方案:

  • 在页面中查找缺失的图片，缺失图片显示有这样的图标 ，要纠正图片地址，双击这个图片，中出来的图片属性对话框中， 输入正确的图片地址。
  • 要删除缺失的图片，点击选中图片标记，然后按退格键或 Delete 键。
  • 如果图片不可用是因为服务器暂时不可用，可以在稍后重试。
  • 如果是缺失了 CSS 文件，您首先要验证 CSS 文件的位置。要在 KompoZer 中修复地址错误， 点击 HTML 源代码标签，在代码方式下编辑文件的位置。如果您对 HTML 标记比较熟悉，可以只编辑 HTML 源代码。

错误消息:

子目录directory name 不存在或没有这个文件名 filename 已经被其他子目录使用

或者

文件名 filename 已经被其他子目录使用

错误描述: 您指定的远程发布站点子目录不存在，KompoZer 只能发布 到一个已经存在的子目录中。或者，您指定的文件名已经有一个同名的子目录使用。

例如，中发布页面对话框的发布标签中:

  • “使用站点子目录”，您可能输入了一个远程站点上，不存在的子目录名称。
  • 如果您选中了“包括图片和其他文件”，然后您输入了一个不存在的子目录名称。
  • 您试图发布的一个文件名，在发布站点上有相同的，已经作为子目录的名称。

解决方案:

  • 使用单独的 FTP 程序，在发布站点上进行创建、重命名或者删除子目录， 问一下服务提供商，如果他们 推荐了一个特定的 FTP 程序，您一般可以在网络提供商网站的帮助和支持里，找到 FTP 的 相关信息，FTP 程序也可以在一些共享站点像 ZDNet 上找到。
  • 子目录的名称不要用“.html”或“.htm”做扩展名，只是 KompoZer 的文件名使用“.html”或“.htm”
  • 子目录的名称是区分大小写的，因此确保输入的子目录名称是精确匹配发布站点上的位置。

错误消息:

服务器不可用，请检查您的网络链接，稍后重试。

错误描述: 这个错误可能由许多原因造成，例如:

  • 您的发布站点设置可能不正确。
  • 您的网络链接可能断开。
  • 您的调制解调器或其他网络连接设备出故障。
  • 发布站点可能出现技术问题，或者发生未知错误。
  • 您的 ISP 或主机服务商可能出现技术问题。

解决方案:

  • 检查您的发布设置是否正确，您的输入也是正确的。更多信息请参阅: 检查您的发布设置。
  • 确保您的网络连接工作正常，尝试用浏览器访问站点，确认您可以成功访问 http://www.mozilla.org。
  • 如果您的网络连接不能工作，检查各个硬件、电话连接、调制解调器是功能正常的。
  • 使用浏览器尝试访问发布站点的某个页面，如果能成功访问其他网站，但不能浏览发布站点里的页面， 那么您的 ISP 或主机提供商可能出现技术问题。
  • 稍后重新尝试发布，您的 ISP 或主机提供商可能暂时出现问题。

错误信息:

您没有发布到这个位置的权限。

错误描述: 您要发布的位置不允许使用，您只能发布到 ISP 或主机提供商允许访问的站点。

解决方案:

  • 检查您在发布站点设置对话框，是否输入了正确的用户名和密码。
  • 联系您的 ISP，查询您可以在哪个站点发布网页。
  • 搜索一个可以发布网页的主机服务网站，在浏览器中，搜索“网站主机”。
  • 使用 KompoZer 我的网页 http://mywebpage.netscape.com/ 发布网页，免费提供 20M 空间。

错误消息:

您当前处于脱机状态，点窗口右下角的图标进行联机。

错误描述: 您正准备发布，但您的 KompoZer 网络连接当前是“脱机”状态，必须是“联机”状态(连接到互联网)才能发布页面。

通过右下角的联机/脱机图标，检查您的网络连接，如果当前脱机状态，会显示 一个图标: 。

解决方案:

  • 点击联机/脱机图标连接网络，联机状态的图标应该这样显示: 。
  • 通过浏览器访问网页，确保您的网络连接是工作正常的，例如，您可以访问这一页来确认: http://www.mozilla.org。

错误信息:

没有足够的可用空间来保存文件 filename。

错误描述: 远程站点磁盘空间已满，或者您可以使用的空间已用完。

解决方案:

  • 使用单独的 FTP 程序，在发布站点上删除不需要的文件子目录， 问一下服务提供商，如果他们 推荐了一个特定的 FTP 程序，您一般可以在网络提供商网站的帮助和支持里，找到 FTP 的 相关信息，FTP 程序也可以在一些共享站点像 ZDNet 上找到。
  • 联系您的 ISP 或主机提供商，增加您的可用磁盘空间，或者使用其他的站点。
  • 如果服务器是在您的公司或学校，联系网络管理员，询问是否可以发布到有更多磁盘空间的位置。 或者您请求增加您当前发布位置的可用磁盘空间。

错误信息:

文件名或子目录名太长。

错误描述: 您要发布的文件名或子目录名太长，超过了站点服务器支持的字符书。

解决方案:

  • 减少文件名或子目录名少于32个字符，一些操作系统不支持超过32个字符的文件名。
